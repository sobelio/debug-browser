  <action>
In `src/main.rs`, build the CLI using clap's derive API:

```
#[derive(Parser)]
#[command(name = "debug-browser", about = "React debugging browser", version)]
struct Cli {
    #[command(subcommand)]
    command: Commands,

    /// Output format
    #[arg(long, default_value = "text", global = true)]
    format: OutputFormat,

    /// Enable verbose logging
    #[arg(short, long, global = true)]
    verbose: bool,
}
```

Define `OutputFormat` enum: `Text`, `Json` (using clap's ValueEnum).

Define `Commands` enum with placeholder subcommands that map to future phases:
- `Navigate { url: String }` — Phase 3
- `Click { selector: String }` — Phase 4
- `Type { selector: String, text: String }` — Phase 4
- `Components` — Phase 7 (list React component tree)
- `Hooks { component: String }` — Phase 8 (inspect hooks for component)
- `Console { action: ConsoleAction }` — Phase 5 (logs/errors inbox)
- `Eval { expression: String }` — Run JS expression

Each subcommand should be a stub that prints "Not yet implemented: [command name]" — actual implementation comes in later phases.

Create command module files (navigate.rs, inspect.rs, console.rs) as empty placeholders with doc comments describing their future purpose. Register them in commands/mod.rs.

Set up tracing-subscriber in main() with env-filter (RUST_LOG support), initialize before command dispatch.
  </action>
  <verify>`cargo run -- --help` shows all subcommands, `cargo run -- navigate https://example.com` prints stub message, `cargo run -- --format json navigate https://example.com` works</verify>
  <done>CLI shows help with all subcommands, each subcommand runs and prints stub, --format and --verbose flags work globally</done>

<task type="auto">
  <name>Task 1: Initialize Cargo project with core dependencies</name>
  <files>Cargo.toml, src/main.rs, src/lib.rs</files>
  <action>
Run `cargo init --name debug-browser` in the project root.

Edit Cargo.toml to add these dependencies:
- clap = { version = "4", features = ["derive"] } — CLI argument parsing
- tokio = { version = "1", features = ["full"] } — async runtime (needed for CDP WebSocket communication)
- serde = { version = "1", features = ["derive"] } — serialization for structured JSON output
- serde_json = "1" — JSON serialization
- anyhow = "1" — error handling with context
- thiserror = "2" — typed error definitions
- tracing = "0.1" — structured logging
- tracing-subscriber = { version = "0.3", features = ["env-filter"] } — log output with RUST_LOG filtering

Set edition = "2024", add a description and license field.

Create src/lib.rs as the library root (empty for now, will house modules).
Keep src/main.rs minimal — just a `#[tokio::main] async fn main()` that prints the tool name and version using `env!("CARGO_PKG_VERSION")`.
  </action>
  <verify>`cargo build` succeeds without errors, `cargo run` prints version info</verify>
  <done>Project compiles, binary runs, all dependencies resolve</done>
</task>

<task type="auto">

  <files>src/lib.rs, src/browser/mod.rs, src/commands/mod.rs, src/inbox/mod.rs, src/react/mod.rs, src/error.rs</files>
  <action>
Create the module directory structure that subsequent phases will populate:

- `src/browser/mod.rs` — Browser connection, navigation, DOM interaction (Phases 2-4)
- `src/commands/mod.rs` — Command registry and dispatch (Phase 9)
- `src/inbox/mod.rs` — Console/error inbox (Phase 5)
- `src/react/mod.rs` — React DevTools hook, component tree, hook inspection (Phases 7-8)
- `src/error.rs` — Error types using thiserror

In `src/error.rs`, define a basic `DebugBrowserError` enum with thiserror derives and a few initial variants:
- `Browser(String)` — browser connection errors
- `Command(String)` — command execution errors
- `React(String)` — React inspection errors

In `src/lib.rs`, declare all modules as `pub mod`. Each module's `mod.rs` should have a brief doc comment and be otherwise empty (placeholder for future phases).

Do NOT add any implementation beyond module declarations and the error enum — keep it minimal.

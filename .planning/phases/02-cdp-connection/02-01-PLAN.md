  <decision>Select CDP implementation approach</decision>
  <context>Phase 2 requires a CDP transport layer. Discovery evaluated three options and studied agent-browser's actual architecture (Playwright + Node.js daemon, NOT raw CDP from Rust). This choice affects all subsequent phases.</context>
  <options>
    <option id="chromiumoxide">
      <name>chromiumoxide crate (recommended)</name>
      <pros>Fully typed CDP bindings for ALL domains, async/tokio-native, built-in browser launch + connect-to-existing, handles WebSocket transport internally, Page.addScriptToEvaluateOnNewDocument is a first-class typed command, actively maintained (v0.9.1, Feb 2026), event_listener<T> for typed CDP event streams</pros>
      <cons>Heavy compile time (~60K generated CDP type lines), large dependency tree</cons>
    </option>
    <option id="raw-websocket">
      <name>Raw WebSocket (tokio-tungstenite)</name>
      <pros>Minimal deps, fast compile, total control over CDP transport, ~300 lines of code</pros>
      <cons>Must build message correlation, event routing, reconnection ourselves, no typed CDP bindings (all raw JSON), must maintain our own CDP types, agent-browser doesn't actually use this pattern (it uses Playwright)</cons>
    </option>
  </options>

<task type="auto">
  <name>Task 2: Create Browser wrapper struct with launch + connect</name>
  <files>src/browser/session.rs, src/browser/mod.rs, src/error.rs</files>
  <action>
Create src/browser/session.rs with a Browser struct that wraps chromiumoxide.

Study agent-browser's BrowserManager in `/Users/whn/tmp/agent-browser/src/browser.ts` — it has launch(), close(), and isLaunched() patterns. Our Rust equivalent:

```
pub struct Browser {
    browser: chromiumoxide::Browser,
    // Handler must be driven — store the JoinHandle so we can abort on drop
    handler_handle: tokio::task::JoinHandle<()>,
}
```

Implement:

1. Browser::launch(headless: bool) -> Result&lt;Self&gt;
   - Call find_chrome() to get the binary path
   - Build BrowserConfig:
     - chrome_executable(path)
     - If headless: headless mode (use .new_headless_mode() for --headless=new)
     - Standard args: --no-first-run, --no-default-browser-check, --disable-extensions, --disable-gpu
   - Call chromiumoxide::Browser::launch(config).await
   - Spawn the Handler in a tokio task: tokio::spawn(async move { while handler.next().await.is_some() {} })
   - Return Browser wrapping the chromiumoxide Browser + handler JoinHandle

2. Browser::connect(url: &str) -> Result&lt;Self&gt;
   - Call chromiumoxide::Browser::connect(url).await
   - Spawn handler same as launch
   - Return Browser

3. Browser::new_page(&self, url: &str) -> Result&lt;chromiumoxide::Page&gt;
   - Delegate to self.browser.new_page(url).await

4. Browser::close(self) -> Result&lt;()&gt;
   - Close the browser connection
   - Abort the handler task

Add BrowserLaunchFailed(String) and CdpError(String) error variants to DebugBrowserError.

Implement Drop for Browser (or a close method) to abort the handler task and ensure cleanup.

Re-export Browser from browser/mod.rs as the primary public API.

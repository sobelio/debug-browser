  <files>src/main.rs, daemon/src/browser.ts, daemon/src/types.ts, daemon/src/actions.ts</files>
  <action>
Add support for connecting to an already-running Chrome instance, following agent-browser's pattern from `/Users/whn/tmp/agent-browser/cli/src/main.rs:269-347`.

In the CLI (src/main.rs):
- Add `--connect <port-or-url>` global flag
- When --connect is set, the launch command sent to daemon includes `cdpUrl` or `cdpPort`

In the daemon:
- BrowserManager.launch() already supports cdpUrl/cdpPort (from agent-browser's browser.ts)
- If cdpUrl is provided, use `chromium.connectOverCDP(cdpUrl)` instead of `chromium.launch()`
- If cdpPort is provided, construct `http://localhost:{port}` and connect

This enables attaching to a dev server's Chrome instance (e.g., `chrome --remote-debugging-port=9222`).

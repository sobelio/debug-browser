2. Launch Browser headless: Browser::launch(true).await

<task type="auto">
  <name>Task 2: Add --connect flag for attaching to existing Chrome</name>
  <files>src/main.rs</files>
  <action>
Add a global --connect flag to the CLI, following agent-browser's --cdp flag pattern from `/Users/whn/tmp/agent-browser/cli/src/main.rs:269-347`.

Agent-browser accepts --cdp with either a port number or a full URL:
```
agent-browser --cdp 9222 navigate https://example.com
agent-browser --cdp ws://localhost:9222/devtools/... navigate https://example.com
```

Our equivalent:
```
debug-browser --connect 9222 navigate https://example.com
debug-browser --connect http://localhost:9222 navigate https://example.com
```

Add to Cli struct:
```rust
/// Connect to existing Chrome at CDP port or URL (e.g., 9222, http://localhost:9222)
#[arg(long, global = true)]
connect: Option<String>,
```

In the navigate handler (and future commands):
- If --connect is set: parse as port number or URL, call Browser::connect()
- If not: call Browser::launch(true)

For port-only values (e.g., "9222"), construct "http://localhost:{port}" — chromiumoxide's Browser::connect() accepts HTTP URLs and auto-discovers the WebSocket URL from /json/version.

This enables debugging already-running apps without relaunching Chrome.

  <action>
Create flake.nix at the repo root with three derivations composed together:

**Inputs:**
- nixpkgs (nixpkgs-unstable)
- crane (github:ipetkov/crane)
- flake-utils (github:numtide/flake-utils)

**Derivation 1 — daemon:**
```
pkgs.buildNpmPackage {
  pname = "debug-browser-daemon";
  version = "0.1.0";
  src = ./daemon;
  npmDeps = pkgs.importNpmLock { npmRoot = ./daemon; };
  npmConfigHook = pkgs.importNpmLock.npmConfigHook;
  # DO NOT set dontNpmBuild — we need npm run build (tsc + copy scripts)
  installPhase = copy dist/, node_modules/ (production only), package.json to $out/lib/daemon/;
}
```

**Derivation 2 — cli (Rust):**
```
craneLib.buildPackage {
  src = craneLib.cleanCargoSource ./.;
  cargoArtifacts = craneLib.buildDepsOnly { ... };
  buildInputs = lib.optionals stdenv.isDarwin [ pkgs.libiconv ];
}
```

**Derivation 3 — composed wrapper:**
```
pkgs.symlinkJoin {
  paths = [ cli ];
  nativeBuildInputs = [ pkgs.makeWrapper ];
  postBuild = wrapProgram with:
    --prefix PATH : ${makeBinPath [ pkgs.nodejs ]}
    --set DEBUG_BROWSER_DAEMON_PATH ${daemon}/lib/daemon/dist/daemon.js
    (Linux only):
    --set PLAYWRIGHT_BROWSERS_PATH ${pkgs.playwright-driver.browsers}
    --set PLAYWRIGHT_SKIP_VALIDATE_HOST_REQUIREMENTS true
}
```

**Also add:**
- `packages.default` = the composed wrapper
- `devShells.default` = craneLib.devShell with nodejs

Use `flake-utils.lib.eachDefaultSystem` for cross-platform support (x86_64-linux, aarch64-linux, aarch64-darwin, x86_64-darwin).

IMPORTANT: The daemon buildNpmPackage needs TypeScript available as a build dependency. TypeScript is in daemon/devDependencies, and buildNpmPackage installs devDependencies during the build phase by default, so `npm run build` (which runs tsc) should work.

IMPORTANT: The installPhase for the daemon should only copy production-needed files:
- dist/ (compiled JS + scripts)
- node_modules/ (after npm prune to production deps only, which buildNpmPackage handles)
- package.json (for Node.js module resolution)

Run `nix build` and fix any build issues iteratively. Common issues to watch for:

1. **importNpmLock failures** — If package-lock.json format issues, try `npmDepsHash` approach instead
2. **TypeScript build failures** — If tsc not found during build, may need to adjust how devDeps are available
3. **Missing libiconv on Darwin** — Ensure Darwin-conditional buildInputs
4. **playwright-driver.browsers version mismatch** — If Playwright version in daemon/package.json doesn't match nixpkgs playwright-driver, either pin the npm version or skip browser wrapping and document manual setup

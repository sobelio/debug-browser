<task type="auto">
  <name>Task 2: Add useDebugValue and custom hook grouping to extraction script</name>
  <files>daemon/src/scripts/get-hooks.js</files>
  <action>
Enhance get-hooks.js with useDebugValue support for custom hook labeling:

1. **_debugHookTypes fallback**: If `fiber._debugHookTypes` is available (React 18.3+), use it as the authoritative hook type array. Map React's internal names to our types:
   - "useState" → "state", "useReducer" → "reducer"
   - "useEffect" → "effect", "useLayoutEffect" → "layout-effect", "useInsertionEffect" → "insertion-effect"
   - "useRef" → "ref", "useMemo" → "memo", "useCallback" → "callback"
   - "useContext" → "context", "useDebugValue" → skip (label-only)
   - "useSyncExternalStore" → "sync-external-store"
   - "useTransition" → "transition", "useDeferredValue" → "deferred-value"
   - "useId" → "id"

2. **useDebugValue extraction**: When walking hooks, if the fiber has `_debugHookTypes` and an entry is "useDebugValue", capture the next hook's `memoizedState` as a `debugLabel` property on the preceding custom hook group. If `_debugHookTypes` is NOT available, skip this feature (heuristic detection of useDebugValue is unreliable).

3. **Custom hook grouping**: If the component source uses custom hooks that call useDebugValue, the debugLabel appears as a hook in `_debugHookTypes`. Group sequential hooks that belong to a custom hook by checking if `_debugHookTypes` contains entries that don't map to known React hooks — these indicate custom hook boundaries. Add an optional `debugLabel` field to hook entries.

4. **Additional hook types**: Add identification for hooks that Phase 3 didn't cover:
   - `useId`: memoizedState is a string ID (e.g., ":r0:"). Detect when _debugHookTypes says "useId" or when value matches the `:rN:` pattern.
   - `useTransition`: memoizedState is `[isPending, startTransition]`. Detect via _debugHookTypes or when value is a boolean (isPending).
   - `useDeferredValue`: similar to useState but used for deferred rendering.

Keep the heuristic-based identification as fallback when `_debugHookTypes` is not available. The _debugHookTypes path should be preferred when present.
  </action>

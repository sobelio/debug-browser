  <files>daemon/src/scripts/get-hooks.js</files>
  <action>
Create a new script that extracts ALL hooks for a specific component. The script receives `{ component: string, depth?: number }` options.

**Component lookup:** Walk the fiber tree (reuse the same _fiberRoots/child/sibling traversal pattern from get-component-tree.js) to find the first function/class component whose name matches `options.component` (case-insensitive substring match, same as the --component filter in Phase 3). If multiple matches, return the first. If no match, return `{ error: "Component not found: ..." }`.

**Hook extraction from fiber.memoizedState linked list:**
Walk the linked list via `hook.next`. For each hook node, identify type by structural heuristics:

1. **useState/useReducer**: `hook.queue !== null && hook.queue !== undefined`. Extract `hook.memoizedState` as value. Distinguish useState from useReducer: if `hook.queue.lastRenderedReducer` and its name is NOT "basicStateReducer" (React's internal useState reducer), it's useReducer.

2. **useEffect/useLayoutEffect**: `hook.memoizedState` is an object with `create` (function), `deps` (array or null), `destroy` (function or undefined), and `tag` (number). Identify by checking `typeof hook.memoizedState?.create === 'function' && 'tag' in hook.memoizedState`. Distinguish effect types via tag bitmask:
   - Passive effect (useEffect): `tag & 8` (PassiveEffect = 8)
   - Layout effect (useLayoutEffect): `tag & 4` (Layout = 4)
   - Insertion effect (useInsertionEffect): `tag & 2` (Insertion = 2)
   Extract: deps array (serialized), whether cleanup exists (`!!destroy`), tag.

3. **useRef**: `hook.memoizedState` is an object with exactly one key `current`, and `hook.queue === null`. Extract the `current` value.

4. **useMemo/useCallback**: `hook.memoizedState` is a 2-element array `[value, deps]` where `hook.queue === null`. Distinguish: if `typeof hook.memoizedState[0] === 'function'`, it's useCallback; otherwise useMemo. Extract: the value (or "[Function]" for useCallback), and deps array.

5. **useContext**: Has no `queue`, `memoizedState` is the context value directly. This is hard to distinguish from other hooks. Detect by checking: if `hook.queue === null` AND hook.memoizedState is NOT an object with a `create` property AND NOT an array with deps AND NOT an object with exactly `{current}` — then classify as "context" (best guess). Skip context hooks that have value `undefined` (empty context).

6. **useDebugValue**: Does not appear in the hook linked list (it's a dev-only label). Instead, check `fiber._debugHookTypes` array if available (React 18.3+) for accurate type identification. If `_debugHookTypes` exists, use it as the authoritative type source instead of heuristics.

**Safety:** Max 50 hooks per component. Use safeSerialize (copy the function from get-component-tree.js) for all values. Default serialize depth 3.

**Return format:**
```json
{
  "component": "ComponentName",
  "type": "function",
  "hookCount": 5,
  "hooks": [
    { "index": 0, "type": "state", "value": "hello" },
    { "index": 1, "type": "effect", "deps": [1, "foo"], "hasCleanup": false, "subtype": "passive" },
    { "index": 2, "type": "ref", "current": null },
    { "index": 3, "type": "memo", "value": 42, "deps": [1, 2] },
    { "index": 4, "type": "callback", "deps": ["x"] }
  ]
}
```

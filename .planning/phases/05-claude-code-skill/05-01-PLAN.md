2. `react detect` — Checks if React is on the page
3. `components` — Lists React component tree (flags: --depth, --include-host, --no-props, --no-state, --props-depth, --component)
4. `hooks <component>` — Inspects hooks for a named component (flags: --depth)
5. `console logs` — Shows collected console.log messages
6. `console errors` — Shows collected errors (console.error + uncaught)
7. `console clear` — Clears the console inbox
8. `click <selector>` — Clicks element by CSS selector
9. `type <selector> <text>` — Types text into input element
10. `eval <expression>` — Evaluates JavaScript in page context
11. `close` — Closes browser and shuts down daemon

Global flags section:
- `--format text|json` (default: text)
- `--session <name>` (default: "default")
- `--connect <port|ws-url>` (attach to existing Chrome)
- `--verbose` / `-v`

Format for LLM consumption: use consistent structure, code blocks for syntax, tables for flags. Avoid prose — be terse and scannable.

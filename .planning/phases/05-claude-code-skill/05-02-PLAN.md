  <name>Task 1: Create references/workflows.md with example debugging scenarios</name>
  <files>skill/references/workflows.md</files>
  <action>
Write 4-5 concrete debugging workflow examples that Claude can follow as templates. Each workflow should show the complete command sequence with expected observations.

**Workflow 1: "Why is my component not rendering the right data?"**
- Navigate to app URL
- Get component tree, filter to target component
- Inspect hooks to see useState/useEffect values
- Identify stale state or missing dependency
- Diagnosis pattern

**Workflow 2: "My app is throwing errors in the console"**
- Navigate to app URL
- Check console errors
- Inspect component tree for error boundaries
- Use eval to check specific values
- Clear console, reproduce, re-check

**Workflow 3: "Component re-renders too often"**
- Navigate, inspect hooks for a component
- Look at useEffect deps, useMemo deps
- Check if callbacks are recreated (useCallback deps)
- Identify missing memoization

**Workflow 4: "Debug a form submission flow"**
- Navigate to form page
- Inspect form component hooks (state values)
- Type into fields, re-inspect state
- Click submit, check console for errors
- Inspect state after submission

**Workflow 5: "Connecting to an existing dev server"**
- Use --connect flag to attach to Chrome with remote debugging
- Then same inspection flow

Format: Each workflow as a numbered step sequence with the exact `debug-browser` command and what to look for in the output. Keep it terse — these are command recipes, not tutorials.

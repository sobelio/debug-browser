<task type="auto">
  <name>Task 1: Create fiber tree walker and add components daemon action</name>
  <files>daemon/src/scripts/get-component-tree.js, daemon/src/types.ts, daemon/src/protocol.ts, daemon/src/actions.ts</files>
  <action>
Create `daemon/src/scripts/get-component-tree.js` — a script that `page.evaluate()` will run to walk the fiber tree and return a serializable component tree.

The script should be a function (or IIFE) that:

1. Access `window.__REACT_DEVTOOLS_GLOBAL_HOOK__`
2. Get fiber roots: iterate `hook._fiberRoots` → for each renderer ID, get the Set of FiberRoot objects
3. For each FiberRoot, get `root.current` (the root fiber / HostRoot)
4. Walk the fiber tree using `child` (first child) and `sibling` (next sibling):
   - Skip internal React fibers (HostRoot, HostPortal, Mode, etc.) — only include fibers where `fiber.type` is a function or class (i.e., user components)
   - For each component fiber, extract:
     - `name`: `fiber.type.displayName || fiber.type.name || 'Anonymous'`
     - `key`: `fiber.key` (may be null)
     - `type`: 'function' or 'class' (check `fiber.type.prototype?.isReactComponent` for class)
     - `depth`: nesting level (for indentation)
     - `children`: recursive array of child component nodes
   - Also include host elements (div, span, etc.) but mark them as `type: 'host'` with `name` = tag name. Include an option/flag to filter these out (default: show only components, not host elements).

5. Return `{ roots: [...], componentCount: N }` where each root is a tree of component nodes.

Add to daemon:
- **types.ts**: Add `ComponentsCommand` (`action: 'components'`, optional `depth?: number` for max depth, optional `includeHost?: boolean`). Add to Command union.
- **protocol.ts**: Add `componentsSchema`. Add to discriminated union.
- **actions.ts**: Add `handleComponents` that reads the script file content, calls `page.evaluate(scriptContent)`, returns the tree. Add to switch.

For reading the script: use `fs.readFileSync` to read `get-component-tree.js` at module load time (cache it), then pass it to `page.evaluate()`. Alternatively, inline it as a string. Use whichever approach is simpler — the script will be evaluated as a function body via Playwright's evaluate.

Important: The fiber tree walker must handle circular references — fibers have `return` (parent) pointers. Only walk `child` and `sibling`, never `return`. Set a max depth safety limit (default 100) to prevent infinite loops from unexpected fiber structures.
  </action>

Build: `cd daemon && npx tsc && cd .. && cargo build`. Navigate to a React app and test: `cargo run -- navigate https://react.dev && cargo run -- eval '...'` (evaluate the component tree script directly first to verify output structure).
  </verify>
  <done>Components daemon action returns structured tree with component names, types, keys, depth. Handles React and non-React pages gracefully.</done>
</task>

<task type="auto">
  <name>Task 1: Create React DevTools hook injection script and auto-inject on launch</name>
  <files>daemon/src/scripts/react-devtools-hook.js, daemon/src/browser.ts</files>
  <action>
Create `daemon/src/scripts/react-devtools-hook.js` — a self-contained JavaScript script that installs `window.__REACT_DEVTOOLS_GLOBAL_HOOK__` before React loads. The hook must:

1. Set `__REACT_DEVTOOLS_GLOBAL_HOOK__` with:
   - `supportsFiber: true`
   - `inject(renderer)` — stores renderer reference in `hook.renderers` (Map keyed by incrementing ID)
   - `renderers`: a `Map()` to hold injected renderers
   - `_fiberRoots`: a `Map()` — React adds fiber roots here (keyed by renderer ID, value is Set of FiberRoot)
   - `onCommitFiberRoot(rendererID, root)` — React calls this on every commit. Store/update root in `_fiberRoots`.
   - `onCommitFiberUnmount(rendererID, fiber)` — no-op for now (needed so React doesn't error)
   - `_debugBrowser: { detected: false, version: null, rendererCount: 0 }` — metadata object for our detection

2. In `inject(renderer)`: set `_debugBrowser.detected = true`, extract version from `renderer.version`, increment `rendererCount`, store renderer in `renderers` Map.

3. Guard against double-injection: if `__REACT_DEVTOOLS_GLOBAL_HOOK__` already exists, skip (don't clobber React DevTools extension if present).

Then modify `BrowserManager.launch()` in `browser.ts`: after creating the context (but before creating/navigating any page), call `this.context.addInitScript({ path: scriptPath })` where `scriptPath` resolves to the hook script. Use `import.meta.url` + `new URL()` to resolve the script path relative to the module (e.g., `new URL('./scripts/react-devtools-hook.js', import.meta.url)`). Convert the URL to a file path with `fileURLToPath`.

Do NOT use `require` or `__dirname` — the daemon uses ES modules.
  </action>
  <verify>
Build daemon: `cd daemon && npx tsc`. Then test: `cargo build && cargo run -- navigate https://react.dev && cargo run -- eval "JSON.stringify(window.__REACT_DEVTOOLS_GLOBAL_HOOK__?._debugBrowser)"` should return an object with `detected: true` and a version string.
  </verify>
  <done>Hook script exists, daemon auto-injects on launch, navigating to a React site shows `_debugBrowser.detected: true` with version info via eval.</done>
</task>

<task type="auto">
  <name>Task 2: Add react-detect daemon action with types and protocol validation</name>
  <files>daemon/src/types.ts, daemon/src/protocol.ts, daemon/src/actions.ts, src/commands.rs, src/main.rs</files>
  <action>
Add a `react-detect` action to the daemon:

1. **types.ts**: Add `ReactDetectCommand` interface (`action: 'react-detect'`).  Add it to the `Command` union.

2. **protocol.ts**: Add `reactDetectSchema` with `action: z.literal('react-detect')`. Add to `commandSchema` discriminated union.

3. **actions.ts**: Add `handleReactDetect` that:
   - Calls `page.evaluate()` to read `window.__REACT_DEVTOOLS_GLOBAL_HOOK__._debugBrowser`
   - Returns `{ detected: boolean, version: string | null, rendererCount: number }`
   - If hook doesn't exist (e.g., page navigated before hook injection), return `{ detected: false, version: null, rendererCount: 0 }`
   - Add the case to the `executeCommand` switch.

4. **Rust CLI**: Add a `React` subcommand group to main.rs with `detect` subcommand. Add `react_detect()` command builder in commands.rs that sends `{ action: "react-detect" }`. Wire it in build_command.

This gives us a dedicated command to check React status rather than relying on raw `eval`.
  </action>
  <verify>
Build both: `cd daemon && npx tsc && cd .. && cargo build`. Test: `cargo run -- navigate https://react.dev && cargo run -- react detect` should return React detection info with detected=true and a version string. Test on non-React site: `cargo run -- navigate https://example.com && cargo run -- react detect` should return detected=false.
  </verify>
  <done>`react-detect` action exists in daemon with full type/validation chain. Rust CLI has `react detect` subcommand. Returns accurate detection for React and non-React sites.</done>

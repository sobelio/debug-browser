Create tests/run-tests.sh — a bash test harness that:

**Setup (run once before all tests):**
1. Build the Rust CLI: `cargo build --release` (skip if binary exists and is newer than source)
2. Install fixture deps: `cd tests/fixture && npm install` (skip if node_modules exists)
3. Build fixture: `cd tests/fixture && npm run build`
4. Start fixture preview server in background: `cd tests/fixture && npm run preview &` — capture PID
5. Wait for fixture server to be ready: poll `curl -s http://127.0.0.1:5188` until 200 (max 10s, then fail)
6. Set CLI path: `CLI="./target/release/debug-browser"`

**Test framework (simple bash assertions):**
```bash
TESTS=0; PASSED=0; FAILED=0

assert_contains() {
  local output="$1" expected="$2" test_name="$3"
  TESTS=$((TESTS + 1))
  if echo "$output" | grep -q "$expected"; then
    PASSED=$((PASSED + 1))
    echo "  ✓ $test_name"
  else
    FAILED=$((FAILED + 1))
    echo "  ✗ $test_name"
    echo "    Expected to contain: $expected"
    echo "    Got: $(echo "$output" | head -5)"
  fi
}

assert_json_field() {
  local output="$1" field="$2" expected="$3" test_name="$4"
  # Uses grep/sed for simple field extraction, no jq dependency
  ...
}
```

**Teardown (always runs, even on failure):**
1. Run `$CLI close` to shut down daemon
2. Kill fixture preview server PID
3. Print summary: "X/Y tests passed"
4. Exit with code 1 if any failures

Use `trap` to ensure cleanup runs on EXIT/INT/TERM.

Make the script executable: `chmod +x tests/run-tests.sh`

Do NOT add jq as a dependency — use grep/sed for JSON field extraction. Keep it simple.

Add test functions to run-tests.sh for the core inspection commands. Each test runs the CLI binary and asserts on output.

**Test: navigate**
```bash
output=$($CLI navigate http://127.0.0.1:5188)
assert_contains "$output" "Navigated" "navigate returns success message"
```

**Test: react detect**
```bash
output=$($CLI react detect)
assert_contains "$output" "React" "react detect finds React"
# Also test JSON format
json=$($CLI --format json react detect)
assert_contains "$json" '"detected"' "react detect JSON has detected field"
assert_contains "$json" '"success":true' "react detect JSON success"
```

**Test: components (full tree)**
```bash
output=$($CLI components --depth 3)
assert_contains "$output" "App" "components shows App"
assert_contains "$output" "Counter" "components shows Counter"
assert_contains "$output" "TodoList" "components shows TodoList"
assert_contains "$output" "TodoItem" "components shows TodoItem"
assert_contains "$output" "ConsoleDemo" "components shows ConsoleDemo"
```

**Test: components with --component filter**
```bash
output=$($CLI components --component TodoList)
assert_contains "$output" "TodoList" "filtered components shows TodoList"
assert_contains "$output" "TodoItem" "filtered components shows children"
```

**Test: components with --no-props --no-state**
```bash
output=$($CLI components --no-props --no-state --depth 2)
# Should show names but no props:/state: lines
assert_contains "$output" "App" "structural view shows App"
```

**Test: hooks for Counter**
```bash
output=$($CLI hooks Counter)
assert_contains "$output" "useState" "Counter hooks shows useState"
assert_contains "$output" "0" "Counter useState initial value is 0"
```

**Test: hooks for TodoList**
```bash
output=$($CLI hooks TodoList)
assert_contains "$output" "useState" "TodoList hooks shows useState"
assert_contains "$output" "Buy milk" "TodoList state contains Buy milk"
```

**Test: hooks JSON format**
```bash
json=$($CLI --format json hooks Counter)
assert_contains "$json" '"success":true' "hooks JSON success"
assert_contains "$json" '"hookCount"' "hooks JSON has hookCount"
```

**Test: close**
```bash

# After close, next command should auto-relaunch daemon
# Verify close succeeded
assert_contains "$output" "OK" "close returns OK"
```

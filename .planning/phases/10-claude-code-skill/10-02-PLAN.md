Step 1 — Create the GitHub repository:
```
gh repo create sobelio/debug-browser --public --description "Rust CLI + Claude Code skill for debugging React applications via headless browser" --source . --push
```
If the repo already exists, just add the remote and push:
```
git remote add origin https://github.com/sobelio/debug-browser.git
git push -u origin main
```

Note: The current branch is `master`. If the remote expects `main`, push as:
```
git push -u origin master:main
```

Step 2 — Create and push v1.1 tag:
```
git tag -a v1.1 -m "v1.1: Skill packaging, Nix flake, public release"
git push origin v1.1
```

Step 3 — Create GitHub release with release notes using gh:
```
gh release create v1.1 --title "v1.1 — Shipping" --notes "$(cat <<'NOTES'
Debug Browser v1.1 — first public release.

What's included:
- Rust CLI for React app debugging (component tree, props, state, hooks)
- Node.js daemon with Playwright for browser automation
- Claude Code skill for AI-assisted React debugging
- Install script (scripts/install.sh) for easy setup
- Nix flake for reproducible builds (nix profile install github:sobelio/debug-browser)
- Full test suite

Install:
  git clone https://github.com/sobelio/debug-browser
  cd debug-browser
  ./scripts/install.sh

Or with Nix:
  nix profile install github:sobelio/debug-browser
NOTES
)"
```

If `gh repo create` fails due to auth, create a checkpoint for the user to authenticate with `gh auth login` first.
